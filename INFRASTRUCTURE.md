## Secrets Management

Pattern: `EnvironmentFile=/etc/[service]/secrets.env` in systemd units. Files are `chmod 600`, owned by root, not in git.

`wp-config.php` holds WordPress DB credentials (standard WordPress pattern, not in git).

`shlink-web-client/servers.json` contains the shlink API key in plaintext (world-readable by nginx — acceptable, key only grants shlink access).

MariaDB root password set during `mysql_secure_installation`, not stored on disk.

## File Transfer

Bidirectional transfer between local and EC2 via S3. Bucket: `umt-temp-transfer` (ca-central-1).
EC2 instance role `UMT.NET-SSM-Role` has `s3:GetObject`, `s3:PutObject`, `s3:DeleteObject` on `umt-temp-transfer/*`.
Local AWS CLI credentials require the same permissions on the same bucket.

Pattern:
# local → EC2
`aws s3 cp ~/file.sql s3://umt-temp-transfer/file.sql`
# on EC2
`~/.local/venv/aws/bin/aws s3 cp s3://umt-temp-transfer/file.sql /home/ubuntu/file.sql`
# cleanup
`aws s3 rm s3://umt-temp-transfer/file.sql`

## CI/CD & GitHub Actions

```
git push → GitHub Actions → AWS OIDC → IAM role ec2-github → SSM SendCommand → deploy script on EC2
```

IAM role: `arn:aws:iam::828007040661:role/ec2-github`
Permissions: `ssm:SendCommand`, `ssm:GetCommandInvocation` on instance `i-0e4757a759e2a9991`
Trust policy: `repo:theinvertedform/*:ref:refs/heads/main` (StringLike — covers all repos under theinvertedform)

SSH deploy key: `/home/ubuntu/.ssh/github_deploy` (ed25519, account-level GitHub key)

### Deploy Scripts

Single generic script at `/usr/local/bin/deploy`, chmod 755, root-owned. Takes target path as argument. Called by SSM as root with no login shell — `HOME` and `GIT_SSH_COMMAND` must be set explicitly.

#!/bin/bash
set -e
TARGET="$1"
if [ -z "$TARGET" ]; then
    echo "Usage: deploy <target-path>" >&2
    exit 1
fi
export HOME=/home/ubuntu
export GIT_SSH_COMMAND="ssh -i /home/ubuntu/.ssh/github_deploy -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null"
git config --global --add safe.directory "$TARGET"
cd "$TARGET"
git pull
chown -R www-data:www-data "$TARGET"
chown -R ubuntu:ubuntu "$TARGET/.git"

### Actions Workflow

```yaml
name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
    steps:
      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::828007040661:role/ec2-github
          aws-region: ca-central-1
      - run: |
          COMMAND_ID=$(aws ssm send-command --instance-ids i-0e4757a759e2a9991 --document-name AWS-RunShellScript --parameters 'commands=["/usr/local/bin/deploy /var/www/TARGET"]' --query Command.CommandId --output text)
          aws ssm wait command-executed --command-id $COMMAND_ID --instance-id i-0e4757a759e2a9991
          aws ssm get-command-invocation --command-id $COMMAND_ID --instance-id i-0e4757a759e2a9991 --query '[StatusDetails,StandardOutputContent,StandardErrorContent]' --output text
```

SSM `--parameters` must be a single line — nested quoting across YAML → bash → AWS CLI → SSM JSON is unreliable. Always use a deploy script; never inline commands in `--parameters`.

**6. nginx server block**
```bash
sudo tee /etc/nginx/sites-available/clientdomain.com << 'EOF'

    listen 80;
```

## Known Issues & Gotchas

### Ownership & Permissions

Multiple actors write to the same directories (SSM as root, git as ubuntu, nginx as www-data).

| Path | Owner |
|---|---|
| `/var/www/*/` files | `www-data:www-data` |
| `.git/` directories | `ubuntu:ubuntu` |
| `secrets.env` | `root:root`, chmod 600 |
| deploy scripts | `root:root`, chmod 755 |

- **git safe.directory** — SSM runs as root; git 2.35.2+ rejects repos owned by another user. Set `safe.directory` explicitly in every deploy script.
- **SSM environment** — no login shell, `$HOME` unset, `~/.ssh/config` not loaded. Always export `HOME` and `GIT_SSH_COMMAND` in deploy scripts.
- **Never `sudo git`** — uses root's SSH config. Fix ownership first, then git as ubuntu.
- **chown `.git/` after every pull** — if `www-data` owns `.git/`, the next git operation as ubuntu fails on `index.lock`.

### pip / Python

Ubuntu 24.04 PEP 668 — no `--break-system-packages`. Use venvs:
```bash
python3 -m venv ~/.local/venv/NAME && ~/.local/venv/NAME/bin/pip install PACKAGE
```
AWS CLI is at `~/.local/venv/aws/bin/aws` — not in root's PATH, never use with sudo.

## Maintenance

```bash
# Health check
systemctl status nginx php8.4-fpm mariadb postgresql@16-main listmonk shlink certbot.timer

# Backup
sudo -u postgres pg_dump listmonk > ~/listmonk-$(date +%Y%m%d).sql
mysqldump -u {clientname} -p {clientname} > ~/{clientname}-$(date +%Y%m%d).sql
cp /var/www/shlink/data/database.sqlite ~/shlink-$(date +%Y%m%d).sqlite
```
